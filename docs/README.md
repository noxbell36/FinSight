## 이 프로젝트를 한 문장으로

> "재무팀에서 직접 수행했던 월마감 비용 검토 업무(법인카드·개인경비 검토, 전표 처리, 월별 비용 증감 분석)를 도구로 만든 것으로, 데이터가 1차 검토를 대신해주고 사람은 판단에 집중하게 하는 **검토 보조 도구**입니다."

## 경험 기반 기능 vs 확장 설계 기능 (중요)

이 구분은 모든 문서와 면접 답변에서 일관되게 유지해야 합니다.

**직접 경험에 기반한 기능** — 실제로 해본 업무를 도구화:
- 월별 비용 실적 집계와 전월 대비 증감 분석 (계정/부서/거래처/적요)
- 전표·경비 검토 7종 (VAT 불일치, 중복 결제 의심, 분할 결제 의심, 적요 미기재, 증빙유형 누락, 건당 고액, 주말 일자 전표)
- 월마감 후 변동 사유 확인·정리 워크플로
- 월간 비용 보고 초안 작성

**확장 설계한 기능** — 직접 담당하지 않았지만, 비용 분석의 자연스러운 다음 단계로 학습·설계:
- 예산 대비 실적 차이 분석 (차이액, 집행률, 초과/미달)
- 연말 착지 전망 (현재 지출 속도 기준 단순 추정)
- 고정비/변동비 성격 분류
- 경영진 보고용 월간 리포트 구조

**아직 범위에 넣지 않은 것** (면접에서 솔직하게):
- 예산 편성(수립) 기능, Forecast, 매출/원가 시뮬레이션 — 실무 경험이 없는 영역이라 "확인하는 기능"까지만 만들고, "수립하는 기능"은 향후 과제로 남김

## 용어 미니 사전

- **판관비**: 판매비와관리비. 이 도구의 분석 대상 비용
- **전월비(MoM)**: (당월 − 전월) ÷ 전월. 전월이 0이면 '신규'로 표시
- **전년동월비(YoY)**: 작년 같은 달과 비교 — 계절성 판단용
- **YTD**: 연초부터 당월까지 누계
- **예산 집행률**: 실적 ÷ 예산. 총계는 예산이 편성된 계정 합계 기준(가중)
- **차이 B/(W)**: Better/(Worse). 예산 − 실적. 음수(초과 지출)가 Worse
- **착지 전망(Landing)**: YTD 실적 + 최근 3개월 평균 × 잔여월 — 연말 예상치의 단순 추정
- **동인(Driver)**: 증감을 실제로 움직인 거래처/부서/적요 — "무엇 때문에 늘었나"의 후보
- **적요**: 전표의 거래 내용 요약 텍스트. 사유 추정의 핵심 단서
