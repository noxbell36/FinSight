# Gemini 활용에 대한 실무적 설명 (Gemini Finance Usage)

## 1. 왜 AI를 썼는가

비용 검토 업무에서 시간이 가장 오래 걸리는 두 구간 — ① 변동 항목의 사유를 문장으로 정리하는 것 ② 보고서 코멘트를 작성하는 것 — 은 "재료는 다 있는데 문장화가 남은" 작업입니다. AI는 정확히 이 구간에 잘 맞습니다. 반대로 숫자 계산과 원인 판단에는 쓰지 않았습니다.

## 2. 역할 분담 — AI가 맡는 것 / 맡지 않는 것

| | 담당 | 이유 |
|---|---|---|
| 집계·증감·집행률·기여도 **계산** | 앱(코드) | 계산은 결정적이어야 하고 매번 같아야 함. AI 계산은 검증 비용이 더 큼 |
| 검토 대상 **선별**, 추세/일회성 **판정** | 앱(규칙 엔진) | 기준이 명시된 규칙이어야 재현·설명 가능 |
| 사유 후보·확인 포인트·코멘트 **문장화** | Gemini (키 입력 시) | 자연어 정리가 AI의 비교우위 |
| 사유 **확정**, 소명 처리, 보고 **책임** | 사람 | 위임 불가 |

핵심 구조: **Gemini는 원본 전표를 직접 해석하지 않습니다.** 앱이 먼저 계산을 끝내고, 그 결과(계정별 실적·증감·집행률, 거래처 기여도, 대표 적요 몇 건, 규칙 엔진 검출 내역)를 구조화된 요약으로 전달하면, Gemini는 그 범위 안에서 문장을 만듭니다. 프롬프트에 "제공된 데이터에 없는 사실 창작 금지, 원인 단정 금지, 수치 변형 금지"를 명시했습니다.

## 3. AI가 실패해도 업무가 멈추지 않는 구조

Gemini 키가 없거나 호출이 실패(한도 초과 등)해도, 모든 화면은 규칙 기반 결과로 완결됩니다 — 변동사유 초안은 규칙 엔진이 거래처 기여도·적요로 생성하고, 리포트의 요약·제안도 계산 기반 문장으로 대체됩니다. AI는 "있으면 더 매끄러운" 레이어이지 의존 대상이 아닙니다. 호출 한도 초과 시에는 일정 시간 대기 후 자동 재시도하며 그동안 추가 호출을 막습니다.

## 4. 왜 원인 확정이 위험한가 / 왜 "사유 후보" 표현인가

데이터에는 "무엇이 얼마나"만 있고 "왜"는 없습니다. 증가분의 72%가 특정 거래처라는 사실이 곧 원인은 아닙니다 — 오분류, 청구 시점 이동, 일회성 정산일 수 있습니다. AI가 "~때문입니다"라고 확정하면 ① 틀렸을 때 보고 사고가 되고 ② 실무자가 검토 없이 복사하게 됩니다. 그래서 출력 전체를 "사유 후보 / ~로 추정, 확인 필요 / 확인 포인트" 톤으로 강제했고, 화면 라벨도 "초안 · 검토 후 사용"으로 통일했습니다.

## 5. 실무자의 AI 결과 검토 방법 (30초 체크)

1. 문장의 숫자가 옆에 병기된 계산값과 일치하는가 (근거 병기는 이 검증을 위한 장치)
2. 언급된 거래처·적요가 실제 전표에 존재하는가 (상세 분석 탭에서 클릭 확인)
3. 데이터에 없는 이야기(계약, 정책 등 외부 맥락)를 만들어내지 않았는가
4. 단정형 문장이 섞였다면 "추정/확인 필요"로 고치거나 삭제

## 6. 자주 받을 4가지 질문에 대한 답

**"AI가 틀린 분석을 하면 어떻게 하나요?"**
→ "틀릴 수 있다는 전제로 설계했습니다. AI는 확정 권한이 없고 초안만 만들며, 모든 문장 옆에 근거 수치가 병기되어 실무자가 즉시 대사할 수 있습니다. 확정은 사람이 버튼으로 하고, 확정 전 문장에는 '검토 전' 라벨이 남습니다. 그리고 AI가 실패해도 규칙 기반 초안이 항상 존재합니다."

**"AI가 만든 코멘트를 그대로 보고해도 되나요?"**
→ "안 됩니다, 그리고 그게 의도입니다. 이 도구의 산출물은 '보고 초안'이고, 수치 대사와 사유 확정을 거친 뒤가 보고 가능한 상태입니다. 도구는 그 검토 진행률을 화면에 보여줘서 '검토를 건너뛰는 것'을 어렵게 만듭니다."

**"재무 데이터에 AI를 쓰는 것이 위험하지 않나요?"**
→ "위험 지점을 두 가지로 나눠 대응했습니다. 첫째 판단 위험 — AI에 계산·판정을 맡기지 않고 문장화만 맡겼습니다. 둘째 데이터 위험 — 이 앱은 서버 없이 브라우저 안에서만 데이터를 처리하고, AI에는 원본 전표 전체가 아니라 계정 단위 집계와 상위 거래처명·대표 적요 요약만 전달합니다. 다만 외부 API로 요약이 전달되는 것은 사실이므로, 실제 회사 도입 시에는 사내 보안 정책 검토가 선행돼야 한다고 답하는 것이 정확합니다."

**"이 프로젝트에서 AI는 정확히 어떤 가치를 더하나요?"**
→ "숫자에서 보고 문장까지의 마지막 구간을 줄여줍니다. 계산과 선별은 규칙 엔진만으로도 되지만, 그 재료를 부서·거래처·예산과 연결해 읽히는 문장으로 만드는 시간이 매달 들었습니다. AI가 그 초안을 만들고 사람이 다듬는 구조로, '판단 대상 찾기'에 이어 '문장 만들기'까지 반복 작업을 줄인 것입니다."

## 7. 면접에서 안전한 표현 / 피할 표현

- 안전: "AI는 앱이 계산한 결과를 문장으로 정리하는 역할입니다" / "원인은 확정하지 않고 사유 후보와 확인 포인트를 제시합니다" / "AI 없이도 규칙 기반으로 동작합니다"
- 위험: "AI가 자동으로 원인을 분석합니다" / "AI가 다 해줍니다" / "정확도가 높아서 믿어도 됩니다" — 검증 구조 없이 신뢰를 말하는 순간, 재무 직무 감각을 의심받습니다.
