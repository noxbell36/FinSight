# FinPilot 고도화 변경 내역

기존 "비용 분석 대시보드"를 **비용 및 예산 대비 실적 분석 도구**로 재구성했습니다.
완전 클라이언트 사이드(A안) 구조로, 서버·DB 없이 브라우저에서만 동작합니다.

---

## 1. 실행 방법

```bash
npm install
npm run dev
```

- 첫 화면에서 **"가상 데이터로 시작하기"** 를 누르면 실적 18개월(2025.01~2026.06) + 예산 2개 버전(본예산·수정예산)이 로드됩니다.
- Gemini AI 기능(변동사유 초안, 매핑 제안, 리포트 요약)을 쓰려면:
  1. https://aistudio.google.com 에서 API 키 발급 (무료)
  2. 앱의 **환경 설정 → Google Gemini API** 에 키 입력 → 연결 테스트
  3. 키는 **이 브라우저의 localStorage에만** 저장됩니다. 코드·저장소·서버 어디에도 저장되지 않습니다.

## 2. 보안 조치 (⚠ 직접 해야 할 일 1건 포함)

| 항목 | 조치 |
|---|---|
| `.env` 커밋 노출 | 파일 삭제 + `.gitignore`에 `.env` 추가 |
| **Supabase anon key 노출** | **직접 재발급 필요** — 아래 절차 참고 |
| Amplitude 세션 리플레이(녹화율 100%) + 네트워크 추적 | `index.html`에서 완전 제거 |
| Supabase Edge Function / Lovable AI 게이트웨이 | 전부 제거 (`supabase/`, `src/integrations/`) — 이제 외부 서버 의존 없음 |

**anon key 재발급 절차** (한 번 노출된 키는 git 히스토리에 남아 있으므로 반드시 재발급):
1. https://supabase.com/dashboard 로그인 → 해당 프로젝트 선택
2. Settings → API Keys → `anon` `public` 키 옆 **Rotate**(재발급) 실행
3. 이 프로젝트는 더 이상 Supabase를 쓰지 않으므로, 다른 곳에서도 안 쓴다면 **프로젝트 자체를 Pause/삭제**해도 됩니다 (가장 확실한 방법).
4. GitHub 저장소가 public이었다면 git 히스토리에 키가 남아 있음 — 재발급했다면 히스토리의 옛 키는 무효가 되므로 문제 없습니다.

## 4. 컬럼 매핑 고도화

- **헤더 행 위치 지정** (제목·결재란이 있는 실무 엑셀 대응) + 자동 감지
- **다중 시트** 선택
- 매칭 엔진 교체: 정규화 완전일치(자동) → 포함일치(추정—확인 표시), `합계금액` 중복 별칭 버그 수정
- **매핑 프로파일 저장 → 동일 양식 자동 적용** (매달 같은 ERP 다운로드 파일 대응)
- **반영 전 검증 리포트**: 반영 예정/빈 행 제외/금액·날짜 변환 실패 행 목록/귀속월 범위
- 금액 단위(원/천원), 귀속월 결정 방식(전표일자 파생/직접 지정)
- 예산은 **세로형 + 가로형(1월~12월 컬럼 자동 인식)** 모두 지원, 버전 지정
- Gemini 매핑 제안(선택): 미지정 컬럼만 헤더+샘플값으로 제안받고 "추정" 표시

## 5. 데이터 구조

- **귀속월(period) 차원** 도입 — 전월비 12.4% 하드코딩 제거, 실제 MoM/YoY/YTD 계산
- 예산: 계정×월×**버전** 구조 (향후 FCST/시뮬레이션 버전 추가 확장 가능)
- **IndexedDB 영속화** — 새로고침해도 데이터 유지 (기존: 휘발)
- 평균 집행률 → **가중 집행률**(총실적÷총예산)로 교정
- "거래 5건 이상 = 증가 추세" 등 근거 약한 판정 제거

## 6. AI(Gemini) 사용 원칙

- 모든 AI 기능은 **수동 실행 버튼** (기존: 대시보드 진입 시 자동 호출 → 크레딧 소모)
- 결과는 항상 "초안 — 검토 필요" 라벨, 계산에는 미사용
- 브라우저 → Google API 직접 호출 (중간 서버 없음)

## 7. 디자인

- 라이트 모드 고정, S-Core Dream + JetBrains Mono 유지
- glow/scan-line/그라데이션/`zoom:0.75` 핵 제거 → 절제된 카드·그림자 1단계
- 재무 표기 관행: 표는 전체 자릿수+콤마, 음수 (괄호), 감소 △, `tabular-nums` 자릿수 정렬, KPI만 억/만 축약
- 이모지 아이콘 제거, 실무 용어("확인 필요 항목", "B/(W)", "소명 필요" 등)

## 9. 참고

- 데모 데이터에는 검토 기능 시연용 이상 항목이 **의도적으로** 심어져 있습니다: VAT 불일치 3건, 중복 결제 1쌍(2026-05 접대비), 분할 결제 1세트(2026-06 소모품비), 주말 전표 2건, 적요 미기재 약 4%, 지급수수료·클라우드 예산 초과 추세.
- 이 도구는 브라우저 로컬에만 데이터를 저장하므로, 회사 실데이터를 사용해도 외부로 전송되지 않습니다. 단, 공용 PC 사용 시 데이터 관리 탭에서 초기화하십시오. AI 기능 사용 시에는 해당 계정·금액·적요 정보가 Google Gemini API로 전송된다는 점을 유의하십시오.

# v2.1 — 인사이트·리포트 고도화 (2026-07-07)

## 핵심 변경: "AI 1차 완결 분석 → 사용자 2차 검토" 구조로 전환
- **월 선택 시 완전 자동 실행**: 월/예산버전 변경 시 규칙 엔진(즉시) + Gemini 배치 분석(1회 호출)이 자동으로 돌고, 결과는 데이터 지문 기반 캐시에 저장되어 같은 데이터로는 재호출하지 않습니다.
- **규칙 엔진(insightEngine)**: 예산 초과/경보, 3개월 이동평균 대비 이탈(추세/일회성/계절성 구분), 계단식 수준 변화, 고정·변동비 분류(변동계수), 계정·거래처 집중도, 전표 검토 요약, **연말 착지 전망**(YTD+최근 3개월 속도)을 AI 없이 즉시 계산.
- **AI 배치 파이프라인(aiPipeline)**: Executive Summary + 검토 대상 계정별 (원인 추론 / 권고 액션 / 변동사유 초안)을 한 번에 생성.

## 화면 변경
- 월별 현황: 상단에 **월간 브리핑(AI 요약 + 인사이트 카드) 상시 노출**, KPI 스파크라인, **전월비 증감 워터폴**, 부서 도넛 복원
- 차이분석·변동사유: **전 항목 펼침 + AI 초안 자동 채움**, 원인/권고 표시, 전체 일괄 확정 버튼. 확정 사유는 월간 리포트에 반영
- 월간 리포트: 8개 섹션으로 재설계 — 요약(AI 코멘트 자동), 예산·실적 콤보+워터폴 차트, 차이 상위, **변동사유(미확정은 AI 초안으로 자동 채움 — 빈 보고서 방지)**, 비용 구조(고정비 비중·성격 분류·집중도), 전표 검토 요약(내부통제), 리스크·권고, 연말 착지 전망
- 예산 대비 실적: 월별 실적 vs 예산 콤보차트 추가
- 전표·경비 검토: **검출 있는 검사 기본 펼침**, 일별 지출 분포 차트 추가

## 타이포그래피
- 숫자 폰트: JetBrains Mono(점 있는 0) → **Pretendard + tabular-nums** (민짜 0, 한글과 톤 통일)
- 한국어 줄바꿈: `word-break: keep-all` 적용 (단어 중간 잘림 해결)
- 전역 크기 1단계 축소(15px 기준), 숫자 자간 정상화
